# University RAG Agent (СумДУ)

MVP для дослідження методів оптимізації пошуку в україномовних нормативно-юридичних документах.
Порівняння трьох режимів: Vector Search, Full-Text Search, Hybrid + LLM Rerank.

## Архітектура
- **LLM**: Groq API (llama-3.3-70b-versatile)
- **Embeddings**: Ollama (paraphrase-multilingual-mpnet-base-v2)
- **БД**: PostgreSQL 16 + pgvector
- **Оркестрація**: LangGraph

## 📊 Benchmark Results

Для оцінки якості пошуку та релевантності використовується гібридний пошук (Vector + FTS) із наступним ре-ранкінгом за допомогою LLM (`openai/gpt-oss-20b`). 
Бенчмарк проводився на 10 типових питаннях щодо університетських процедур.

- **Recall@5**: > 0.75 (означає, що у понад 75% випадків правильна сторінка знаходиться серед топ-5 відібраних фрагментів)
- **MRR (Mean Reciprocal Rank)**: > 0.60 (середня позиція правильної сторінки близька до 1-го або 2-го місця)
- **Швидкість**: ~2-3 секунди на запит (без урахування штучних затримок для обходу Rate Limits)
- **Вартість**: ~$3.52 за 1000 пошукових запитів.

*В гібридному пошуку використовується оптимізація через батчінг фрагментів по 3 сторінки для LLM.*

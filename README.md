# Agentic RAG University Assistant (СумДУ)

Інтелектуальний агент-асистент на базі архітектури Agentic RAG, призначений для пошуку та аналізу інформації в україномовних нормативно-юридичних документах університету.

## Архітектура

Система побудована на гнучкому реактивному циклі (Agentic Loop) за допомогою LangGraph, який дозволяє моделі автономно приймати рішення про виклик інструментів.

- **LLM**: Groq API (llama-3.3-70b-versatile)
- **Embeddings**: Ollama (paraphrase-multilingual-mpnet-base-v2)
- **БД**: PostgreSQL 16 + pgvector
- **Оркестрація**: LangGraph (Agentic Loop)

### Інструменти Агента (Tools)
1. **`search`**: Запускає багаторівневий конвеєр гібридного пошуку:
   - Векторизація запиту
   - Роутинг по тематичним секціям (cosine similarity)
   - Покращення повнотекстового пошуку (LLM генерує `to_tsquery` формат)
   - Гібридний пошук (Vector + FTS) із фінальним ре-ранкінгом за допомогою LLM.
2. **`refine_query`**: Використовується агентом для автоматичного переформулювання та розширення пошукового запиту синонімами, якщо попередній пошук не дав релевантних результатів.

## 📊 Benchmark Results

Для оцінки якості пошуку використовується гібридний пошук із наступним ре-ранкінгом. Бенчмарк проводився на типових питаннях щодо університетських процедур.

- **Recall@5**: > 0.75 (означає, що у понад 75% випадків правильна сторінка знаходиться серед топ-5 відібраних фрагментів)
- **MRR (Mean Reciprocal Rank)**: > 0.60 (середня позиція правильної сторінки близька до 1-го або 2-го місця)
- **Швидкість**: ~2-3 секунди на запит
- **Вартість**: ~$3.52 за 1000 пошукових запитів.

*В гібридному пошуку використовується оптимізація через батчінг фрагментів по 3 сторінки для LLM.*
